Budget Feasible Mechanism on Matroids

1. Background

本文主要是基于拟阵设定情形下的预算可行性机制的设计；拟阵通常是由一个二元组来表示$\mathcal{M}=(E,\mathcal{I})$，$E$表示所有元素element的全集，而$\mathcal I\subseteq 2^E$是由$E$的子集组成的集族，对于$I\in \mathcal I$称之为独立集（independent set）；拟阵有两种性质：

  • Hereditary property：如果$I\in \mathcal I$，且$j\subset I$，那么$j\in \mathcal I$；
  • Exchange property：对于$I,J\in \mathcal I$，如果$|I|<|J|$，那么存在一个元素$e\in J$满足$I\cup {e}\in J$；

现有一个buyer有$b$的预算，他想要在独立集中尽可能的购买比较多的value值，同时它的value function是一个线性加和的函数；假设元素$e$的权重为$w_e$，私有成本cost为$d_e$，用$bb(e)=\frac{d_e}{w_e}$表示buck-per-bang；

2. Mechanism

（1）首先考虑单个拟阵问题，分析一下问题，介绍算法设计的intuition；主要包含两点：

  • 一是：假设现在有一个element，它的weight很大，任何一个solution如果不包含这个item的话，都会造成一个很差的解（相对于最优解而言）；为了解决这样的问题，我们首先将最大weight的元素去掉，然后从剩下的element组成的独立集中找到一个解，然后将这个解的value总值与这个最大weight进行比较，最大的那一个作为最终的输出；
  • 二是：在budget feasible mechanism中，通常最终的payment都是采用proportional scheme，即对于每一单位weight都会有一个统一价格，而且会采用单调的选择方式，可以保证truthfulness，因此我们可以从一个较高的价格开始，若是在这个价格下满足的元素总付费超过预算，就降低价格，知道满足budget限制为止；

这两点就构成了算法，同时证明该算法保证4-competitive ratio；

（2）现在考虑多个拟阵，Mechanism for matroid intersections；假设现在有$k$个拟阵$\mathcal M_1,…,\mathcal M_k$，将其转化为一个真实的拟阵，$\mathcal M=(E,\mathcal I)$，此时$\mathcal I=\cap_j \mathcal I_j$；需要说明的是想要得到$\mathcal I=\cap_j \mathcal I_j$也是一个NP-hard问题；所以我们先用一个多项式时间 blackbox APX得到一个$\alpha-$近似的结果，然后后续算法与单拟阵的算法类似；

3. XOS function

在budget feasible mechanism中，线性加和函数是最简单的value function，可以考虑更为general的函数，如子模函数，或者是XOS function；

几种常见value function的关系：

Additive function $\subset $ Separable piece-wise linear and concave (SPLC) $\subset$ OXS $\subset $Gross substitutes (GS) $\subset $ submodular $\subset $ XOS,

Additive function $\subset $ Budget additive (BA) $\subset $ submodular $\subseteq $ XOS $\subseteq$ sub-additive;

  2. 以0.5的概率执行一下操作：类似于profit extract method；
    • 将所有的item随机分到两个集合$T_1,T_2$；
