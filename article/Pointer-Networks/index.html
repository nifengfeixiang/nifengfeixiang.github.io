Neural Combinatorial Optimization with Reinforcement Learning

1 TSP Problem

利用强化学习和pointer network解决TSP问题。首先是对TSP描述：表示n个地点，同时每个地点用$\mathbf{x}_{i}$二维坐标表示。那么对于一个路径或者是地点的一个排列序列$\pi$，这个路径的长度可以表示为：

我们想要学习到一个随机策略（stochastic policy）$p(\pi|s)$，给出一个地点集合$s$，输出对一个序列或是路径的概率：长路径概率低，段路径概率高。

如果使用chain rule的方式，通过排序，输入一个sequence，输出一个sequence，这样的方式存在两个问题：一是不能够泛化，二是需要groundtruth来训练。因此采用pointer network来作为策略模型（policy model）。

2 Architecture

Pointer network由两个recurrent neural network（RNN）组成，encoder和decoder，每个都由LSTM组成。

Encoder网络读取序列$s$，每次读取一个地点，并将其转化成一个潜在记忆状态的顺序（sequence of latent memory states）表示为：$\{enc_i\}_{i=1}^{n}$；需要注意，读取的第$i$个地点$\textbf{x}_i$不在是二维坐标，二是一个embedding之后的d-维向量（通过一个所有点共用的线性的转化，linear transform，通常有什么样的线性转化？）

Decoder网络维持一个潜在记忆状态的顺序表示为：$\{dnc_i\}_{i=1}^{n}$；在每一个步骤输出下一个地点的分布，在选择完一个地点后，输入给下一步，再来决定下下个访问地点。需要注意的是，第一个输入$$是一个d-维，需要我们网络来训练的参数。

Attention function（这个函数的作用是什么？可以参考：https://zhuanlan.zhihu.com/p/57672079）：此时query $q=dec_i$，同时对应的reference（Key？）是$ref=\{enc_1,enc_2,…,enc_k\}$。预测一个distribution $A(ref,q)$，表示与$r_i$的相关性程度。

Attention: 就是一种权重参数的分配机制，目标是协助模型捕捉重要信息。具体一点就是，给定一组，以及一个目标（查询）向量query，attention机制就是通过计算query与每一组key的相似性，得到每个key的权重系数，再通过对value加权求和，得到最终attention数值。

3.2 Baseline-critic

上面提到，找到一个合适baseline $b(s)$可以提高训练学习的效果，这里采用critic 网络，参数为$\theta_v$，来学习我们当前策略$p_\theta$下的期望路径长度。评论家接受了随机梯度下降训练，其均方误差目标介于其预测值$b_{\theta_v}(s)$与最新策略采样出来的一个实际长度之间，（这个地方为什么是减L()，而不是J(\theta)，这个地方是从当前策略中sample出一个确定的路径）

Critic architecture：现在来解释critic的构造，主要包含有三个模块：一个LSTM encoder，一个LSTM process block，和一个2-layer ReLU网络decoder。LSTM encoder与pointer network类似结构，输入sequence $s$并将其转为latent memory states以及一个隐藏状态hidden h。然后是处理模块process block，处理块对隐藏状态h执行P个计算步骤。 每个处理步骤都通过glimpsing memory states状态来更新此隐藏状态h，并将瞥见功能的输出作为输入提供给下一个处理步骤。process block结束后，获得的隐藏state被解码为一个baseline的预测结果。
